1. 타입스크립트는 컴파일 시점에 타입 검사를 통해서 기존의 JS 런타임 오류를 잡아준다.
또한 JS보다 많은 기능과 더 효율적인 에러 메시지를 통해 효율적인 코드를 작성할 수 있다.

2. 매개변수에 _을 사용하면 사용하지 않겠으니 에러를 발생시키지 말라는 뜻이다.


3. type
3-1. 타입 별칭을 정의하는데 사용합니다.
3-2. type typeName = 타입;

3-3. type typeName = {key : type, key : type}
=> 객체 타입을 지정할 수 있습니다.

3-4. type typeNmae = 일반 타입

3-5. type typeName =  배열 타입

3-6. type typeName = 타입 | 타입 (유니언 사용 가능)

3-7. type typeName = 타입 & 타입
=> 두 개 이상의 타입을 결합시킵니다. 결합된 타입들을 모두 만족시켜야 값을 허용합니다. (교집합)

3-8. type typeName = (매개변수타입, 매개변수타입...) => 반환 타입
=> 함수 타입

3-9. type typeName<T> = {
    value : T
} => 제너릭 타입

3-10. 조건부 타입

3-11. 리터럴 타입


4. 유니언
4-1. 여러 타입의 결합하며 하나의 타입으로 사용할 수 있게 해줍니다. (합집합)
4-2. 파이프 기호를 사용합니다(|), 여러 타입을 결합 가능, typeof나 instanceof, 조건문을 통해 유니언 타입
좁히기 가능, null과 undefined 허용
4-3. const vari : string | number (문자열 or 숫자)
